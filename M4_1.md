## Schema (v1)

Six tag dimensions per champion. Game-plan affinity is **not** tagged here — it's derived in M4.2 from these dimensions.

| Dimension | Values | Notes |
|---|---|---|
| `damageType` | `AP` \| `AD` | Plus optional `secondaryDamageType` for genuine hybrids (Kog'Maw). |
| `range` | `melee` \| `ranged-short` \| `ranged-long` | AA-bucket default; overrides documented in `notes` per rule A. |
| `engageTool` | `none` \| `soft-single` \| `hard-aoe` | hard-aoe = AoE hard CC initiation locking 2+ enemies. |
| `disengageTools` | array of `peel` \| `knockback` | Empty array = no disengage. |
| `mobility` | `low` \| `medium` \| `high` | Strict count of dashes/blinks (rule B); overrides documented. |
| `scalingWindow` | `early` \| `mid` \| `late` | Plus boolean `hyperCarry` for true 1v9 late-game scalers. |

Plus optional fields: `gamePlanHints` (override hints for M4.2 classifier), `notes` (free-text rationale), `confidence` (`low` \| `medium` \| `high`), `taggedBy` (`claude` \| `user` \| `verified` — workflow state).

## Arbitrated rules (transverse decisions)

These four rules apply to all future tagging work and resolve recurring judgment calls.

### Rule A — Range override for ability-range kits
AA-range bucket is the default. Override to `ranged-long` only when the champion is *primarily played* as a long-range artillery mage. Explicit list: **Cassiopeia, Heimerdinger, Karthus, Ziggs, Vel'Koz, Xerath**. Override is documented in `notes` and uses the criterion "primary lane playstyle is long-range poke/control with kit-effective range >800."

Champions whose only long-range tool is a finishing ult (Caitlyn R, Ashe R) keep their AA-range bucket.

### Rule B — Mobility = strict count + documented overrides
Mobility level is the count of dashes/blinks/teleports in the kit:
- 0 = `low`
- 1 = `medium`
- 2+ = `high`

Charges, ramp-speed effects, and stealth movement do **not** count by default. Overrides allowed when the kit-defining mobility is unambiguously high despite the count rule (Hecarim E + R = `high`); the override must be documented in `notes` so the rule remains reproducible.

### Rule C — Knockback dual-purpose abilities
Abilities that can both engage and disengage (Lee Sin R, Tristana R, Gragas E) are tagged as `disengageTools: ['knockback']` regardless. No `dualPurpose` flag. The M4.2 Game Plan Classifier handles context-dependent reading.

### Rule D — Trap-as-peel
Peel = ability that *actively/reactively* protects an ally from incoming engage. Traps that require an enemy to step on them (Caitlyn W, Teemo R) do **not** count — they are passive zone control, not reactive peel. Active polymorph/silence/root used reactively (Lulu W, Ahri E) does count.
