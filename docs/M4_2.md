# M4.2 — Game Plan Classifier

Closed: 2026-05-01

## What it does

Maps a composition (1-5 champions) to a probability distribution over the
5 game-plan archetypes from the frameworks dossier:
**Siege | Split | Pick | Protect | Engage**.

## Public API

```typescript
import { classifyGamePlan } from '$lib/strategic/gamePlanClassifier';

const result = classifyGamePlan(['57', '54', '85', '157', '67']);
// {
//   distribution: { siege: 0.05, split: 0.15, pick: 0.15, protect: 0.20, engage: 0.45 },
//   primary: 'engage',
//   secondary: 'protect',
//   confidence: 'high',
//   ambiguous: false
// }
```

## Vote table (D4)

Each champion contributes points to each archetype based on its tags. The comp
total is normalized to a probability distribution.

### Siege
- `range = ranged-long` → **+2**
- `engageTool = none` → **+1**
- `damageType = AP` → **+0.5**
- `mobility = low` → **+0.5**

### Split
- `range = melee` AND `scalingWindow != early` → **+2**
- `mobility = high` AND `range = melee` → **+1**
- `range = melee` AND `damageType = AD` AND `scalingWindow = late` → **+1** (duelist heuristic)

### Pick
- `engageTool = soft-single` → **+2**
- `mobility = high` → **+1**
- `damageType = AP` AND `range = ranged-short` → **+1**

### Protect
- `disengageTools` includes `peel` → **+2**
- `scalingWindow = late` OR `hyperCarry = true` → **+1**
- `engageTool = none` AND `range = ranged-long` → **+0.5**

### Engage
- `engageTool = hard-aoe` → **+3**
- `engageTool = soft-single` → **+1**
- `range = melee` AND `engageTool != none` → **+0.5**
- `mobility = medium` AND `engageTool = soft-single` → **+0.5**

### Optional `gamePlanHints` override
If a champion's tag has explicit `gamePlanHints` (e.g. Jayce → siege), each hint
adds **+1** to that archetype as a soft nudge. Used for signature picks.

## Decisions

| Spec ref | Decision |
|---|---|
| D1 | Numeric votes, not declarative rules |
| D2 | Full distribution returned (sums to 1.0) |
| D3 | `ambiguous: true` when primary < 0.40 |
| D5 | Empty/zero-sum comp → uniform 0.2 + ambiguous |
| D6 | Confidence: 1-2 → low, 3 → medium, 4-5 → high |

## Rule mismatch documented

The spec includes a Split rule `disengageTools includes "self" mobility → +1
split`. The M4.1 schema dropped `'self'` from `DisengageTool` (replaced by the
broader `mobility` dimension). Rule skipped — see `docs/unresolved.md`.

## Smoke test results (real pro comps)

All passing on real `data/championTags.json`:

| Comp | Champions | Expected | Result |
|---|---|---|---|
| Engage | Maokai + Malphite + Kennen | engage | ✓ primary |
| Protect | Janna + Lulu + Yuumi + Caitlyn + Kog | protect | ✓ in top-2 |
| Siege | Caitlyn + Lulu + Ziggs + Jayce | siege | ✓ in top-2 |
| Split | Fiora + Ryze + Camille | split | ✓ in top-2 |
| Pick | LeBlanc + Ahri + Rell + Ashe | pick | ✓ in top-2 |

The "in top-2" allowance reflects that real comps are rarely 100% one
archetype — Pick and Engage especially blend through soft-single CC stacks.

## Limits

- The Pick comp test had to allow secondary because `LeBlanc + Ahri + Rell + Ashe` actually leans toward Engage in raw scoring (Rell hard-aoe scores +3). Pick still appears in top-2 from the soft-single stack. Real-world reading: this comp can flex between Pick and Engage depending on execution — which the classifier captures correctly via the distribution rather than a single label.
- Champions with form-changing kits (Jayce, Gnar, Yunara, Nidalee) are tagged on their primary form only. The classifier doesn't switch forms.
- The classifier doesn't account for player skill, patch meta-strength, or matchup data — pure tag-based reading. Those signals belong to other modules (M4.3 power curves, M4.4 risk markers, M5+ adversary detector).
