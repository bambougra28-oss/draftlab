# M4.4 — Risk Markers

Closed: 2026-05-01

## What it does

Inspects an ally composition (and optionally enemy comp + game plans + dataset)
and surfaces strategic risks as actionable markers, sorted critical-first.

## Public API

```typescript
import { detectRiskMarkers } from '$lib/strategic/riskMarkerDetector';

const markers = detectRiskMarkers(
    allyComp,
    enemyComp,
    {
        allyGamePlan,    // GamePlanResult from M4.2
        enemyGamePlan,   // GamePlanResult from M4.2
        dataset          // for matchup lookups
    }
);
// [{ id: 'no-disengage-vs-engage', severity: 'critical', message: '...', rationale: '...' }, ...]
```

## Markers implemented (9 of 12)

| ID | Severity | Trigger |
|---|---|---|
| `no-frontline` | warning | No melee champion that's an engager OR a non-late-scaler |
| `damage-100-ad` | warning | All ally champions pure AD (no `secondaryDamageType`) |
| `damage-100-ap` | warning | All ally champions pure AP (no `secondaryDamageType`) |
| `no-engage-tool` | critical / warning / info | No `engageTool != 'none'`. Severity scales with `allyGamePlan.primary` |
| `no-disengage-vs-engage` | critical | `enemyGamePlan = engage` AND no ally `disengageTools` |
| `difficult-lane-matchup` | warning | Any same-role matchup with WR <45% and games ≥100 |
| `homogeneous-scaling` | info | All champions share `scalingWindow = early` or `late` |
| `low-mobility-vs-pick` | warning | `enemyGamePlan = pick` AND ≥3 ally champions `mobility = low` |
| `split-without-waveclear` | warning | `allyGamePlan = split` AND no `range=ranged-long` + `damageType=AP` champion |

## Markers deferred to M5+ (data layer not yet available)

| ID | Reason |
|---|---|
| `side-disadvantage-unmitigated` | Requires `sideStats` input — not wired through yet |
| `pool-overlap` | Requires `playerPools` input — M5 Pool Tier Classifier territory |
| `pocket-pick-risk-enemy` | Requires `playerPools` + patch data — M5 Pocket Pick Risk Assessor |

These three are listed in the type union (`RiskMarkerId`) so the contract is
forward-compatible. They're skipped silently when the corresponding context
fields are absent.

## Sort order (D15)

`critical` → `warning` → `info`, then alphabetical by `id` for stable output.

## Limits

- Wording is FR-only. No i18n in M4.
- `no-engage-tool` info severity requires `allyGamePlan` set; missing context
  defaults to `warning`. The prototype always passes both game plans.
- `homogeneous-scaling` requires ≥3 champions to fire (avoids noise on
  partial drafts).
- Matchup data quality varies by champion popularity; we filter `n < 100` so
  rare matchups don't flag as difficult.
